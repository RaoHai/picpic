友邻广播显示

采用Memcache与数据库结合的做法 1-所有的数据在Active表，id号递增，时间排序 2-对于每个人，在memcache中维护他最近[1天]的动态（id，type) 列表（合并/过滤都在这个列表上进行） 3- 当需要查看1天前的广播时，直接查db（这个[1天]根据用户数量进行调整） 4-单条广播在memcache中单独为一个key 5-查看友邻广播时，首先拿到所有友邻的广播(id,type)列表，按id排序合并，去掉不能看到的type,然后过滤，合并。 同种action的合并（关注了同一个画集，同加入了一个小组） 同一个人的同种行为合并（消除刷屏的危险）

    • 合并一个人连续的同类型行为（推荐）
    以上 参考自豆瓣。 新增/删除的列表使用差集维护。。
