Installation

Unzip the .zip or .tgz archive. A new directory named 'vogoo' will be created. It contains all the files required for the Vogoo PHP LIB.

Copy this directory to your web site (keeping the full structure of the directory). Change the permissions on the 'config.php' file to make it writeable for all (666 or -rw-rw-rw mode in your FTP client).

Open your Web browser to the location you placed Vogoo PHP LIB in, adding 'install/install.php' to the address, e.g. http://www.mydomain.com/vogoo/install/install.php

A new page with the title 'Welcome to Vogoo LIB installation' should appear. This page contains a form as shown on the image below.

Fill in all the requested information. Choose the Database type of your Hosting Service.
In order to fill in the other fields, check the information given by your hosting service. Your private server hostname, database, username and password should have been sent to you when you created your account.

Click 'Start Install'. A new page informing you about the installation process should appear. If the installation succeeded, read the following section. Otherwise, go to the troubleshooting section below.

Completing the installation and using Vogoo LIB
Once the installation succeeded, change permissions on 'config.php' to make it writeable only for yourself (644 or -rw-r--r-- mode).
To use the Lib, you must include the file 'vogoo.php' (contained in the vogoo directory) in your scripts. A new object, called $vogoo, will be created. The syntax used to call a Vogoo PHP Lib function in your scripts is: $vogoo->functionname()

Important information
Whenever you include the 'vogoo.php' file in your script, a new object named $vogoo is created and a Database connection is opened. If the connection to the Database failed, you won't be able to perform any operation allowed by Vogoo PHP LIB. Therefore, once you included the 'vogoo.php' file, please check $vogoo->connected boolean value. If this is set to true, the connection is opened, otherwise a problem has occured.
We also recommend you to add a constant to your scripts specifying the Vogoo LIB directory. Although Vogoo PHP LIB includes a mechanism to find its installation directory by itself, specifiying it in your scripts is a far more reliable solution. To do so, add the following directive to your script just before the include('vogoo/vogoo.php') line:
define('VOGOO_DIR','[Vogoo installation path]');

Troubleshooting
If a problem occured during the installation, you can try to launch a new installation process after checking a few elements.
First, erase the content of the 'config.php' file. Do NOT erase the file itself but erase its content, if any. Then check the write permissions of the file as specified above.
If the file does not exist, create an empty one with the right permissions.

If you are still unable to use the Lib, go back to the Install page. Check carefully all the information you provide in the form.
It's often the main reason for any failure during the installation.

Upgrading Vogoo PHP LIB
Please check the Download section on vogoo-api.com website if you want to upgrade from a previous version of Vogoo PHP LIB.

Uninstalling Vogoo PHP LIB
Before uninstalling Vogoo PHP LIB, make sure that your scripts do not use on the LIB anymore.
The uninstallation process is not automatic. You must first remove the directory you installed the LIB to. Then connect to your Database (through the interface provided by your Hosting Service or by any other method) and drop the 4 SQL tables called vogoo_ratings , vogoo_ads, vogoo_ads_products , vogoo_links (SQL Syntax: DROP TABLE vogoo_ratings , ...).
